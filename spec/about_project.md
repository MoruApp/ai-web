# Ai Web

## Overview
社内の総務・IT・人事への問い合わせ対応を自動化する「AIコンシェルジュ」Webアプリ（SaaS）のβ版を、AIの自律的な開発能力の検証も兼ねて短期間で構築・社内試験運用するプロジェクト。

## Objectives
- 社内問い合わせ対応コストを30〜40%削減できるレベルの自動応答を実現する
- AIが要件からWebアプリ（UI含む）を自動生成できるかを実証・検証する
- 1か月以内にMVP（β版）を開発し、社内限定で試験運用を開始する

## Scope

### In Scope
- 社内FAQを用いたAIコンシェルジュ Webアプリ（β版）の開発
- 一般社員向け問い合わせUI（PC/スマホ対応）の実装
- 管理画面（FAQ登録・編集、会話ログ閲覧）の実装
- 社内認証との連携によるログイン機能の実装
- 問い合わせ履歴閲覧機能の実装
- AIによる自然言語での回答機能の実装
- 社内限定での試験運用（β版リリース）
- 簡易利用マニュアルの作成
- AIを活用したプロトタイプ生成・コード自動生成プロセスの適用と記録

### Out of Scope
- 社外向けSaaSとしての本番運用・商用リリース
- 多言語対応（日本語以外）[要確認]
- 外部ネットワーク／クラウド環境からのアクセス対応
- 他社システム（勤怠・経費精算など）との直接API連携 [要確認]
- コールセンターなど電話チャネルとの連携
- 高度な分析ダッシュボードやレポーティング機能の実装
- 回答精度の自動改善機能の本格実装（PoCレベルの検証に留める想定）[要確認]

## Key Constraints
- 開発期間は約1か月以内でMVP（β版）を完成させること
- 社内ネットワーク内のみで利用可能とするセキュリティ制約
- レスポンス時間は2秒以内を目標とする性能要件
- AIによる自律的な開発を前提とし、人手による大規模な作り直しは最小限に抑える
- 利用開始時点ではFAQデータが限定的であり、回答精度が段階的に向上する前提での運用開始

## Success Criteria
- 社内問い合わせ対応工数が、対象部門（総務・IT・人事）で平均30〜40%削減されたと評価されること（一定期間の比較に基づく）[要確認：測定方法・期間]
- 一般社員・管理者のテストユーザーのうち、一定割合以上が「業務上有用」と評価すること（例：満足度アンケートで70%以上）[要確認：具体指標]
- AIが要件定義からUIプロトタイプ、コード自動生成までの一連の開発プロセスにおいて、主要機能の実装を自律的に完了したことを示すエビデンスが残っていること
- レスポンス時間2秒以内、社内認証連携、PC/スマホ対応などの非機能要件を満たしていること
- 重大なセキュリティインシデントや業務影響の大きい誤回答が発生せずにβ版運用期間を完了すること

## Background
社内の総務・IT・人事部門には、手続き方法や各種ルールに関する問い合わせが日常的に多数寄せられており、担当者が同様の質問に繰り返し対応することで大きな工数が発生している。一方、汎用的なチャットボットでは、社内固有のFAQや文脈に十分対応できず、実務での活用に限界がある。

この状況を踏まえ、社内FAQを活用した専用の「AIコンシェルジュ」を構築し、問い合わせ対応を自動化することで、担当部門の負荷軽減と社員の自己解決率向上を図る。また、本プロジェクトは単なる業務効率化に留まらず、「AIが要件からWebアプリを自律的に開発できるか」を検証する技術的チャレンジでもあり、プロトタイプ生成、要件検証、UIワイヤーフレーム作成、コード自動生成までのプロセスにAIを積極的に活用する。

プロジェクトは社内限定のβ版リリースをゴールとし、来月末までを目標期間として、WBSに基づき要件整理・プロトタイプ生成・モデル連携・β版開発・テスト・リリースを短期間で実施する計画である。これにより、社内問い合わせ業務の効率化と、今後のAI活用開発手法の社内ナレッジ蓄積を同時に実現することを目指す。
